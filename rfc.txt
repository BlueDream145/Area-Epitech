#### Area ####

	Area est un projet de mise à disposition de services et actions.
	Un serveur est lancé gérant une base de donnée (MySql) et mettant à disposition des clients,
	un certain nombre de fonctionnalités. Les clients sont Mobiles et Web.

	L'architecture du projet est la suivante:

	# Area
	 
		# Server
			Folder = Area/Area.Server
			
		# Mobile Client
			Folder = Area/Area.MobileClient
				~IOS lib: Area/Area.MobileClient.Android
				~Android lib: Area/Area.MobileClient.IOS

		# Web Client
			Folder = Web
			
#### Introduction ####
			
	Le serveur se lance sur le port 8080 et communique en JSON respectant les principes de RESTApi.
	Le serveur web se lance sur le port 8181.

	Nous mettons à disposition, neuf services:

		public enum ServiceEnum
		{
			Test = 0,
			Imgur = 1,

#### Protocole ####
		
	Le protocole fonctionne avec un système de MessageId.
	Chaque données transmise est identifiable via une variable qui définit la structure de la donnée.
	Voici les différents paquets pris en charge par l'application:

		NetworkEnum
		{
			Empty = 0,
			
			// Handshake
			HelloConnectMessage = 1,
			
			// Connection
			IdentificationMessage = 2,
			IdentificationResultMessage = 3,
			RegisterMessage = 4,
			RegisterResultMessage = 5,
			
			// Services
			RegisterServiceMessage = 6,
			DeleteServiceMessage = 7,
			ServiceListMessage = 8,
			ServiceMessage = 9,
			
			// Entities
			ActionMessage = 10,
			ReactionMessage = 11,
			
			// Other
			UnknowBehaviourMessage = 12,
			
			// Profile
			ProfileUpdateRequestMessage = 13,
			ProfileUpdateResultMessage = 14,
			
			// Actions
			ActionRequestMessage = 15,
			ActionResultMessage = 16,
			
			// Reactions
			ReactionRequestMessage = 17,
			ReactionResultMessage = 18,
		}
		
	Les paquets envoyés par le servent sont les suivants:
		
			- IdentificationResultMessage = 3
			- RegisterResultMessage = 5
			- ServiceListMessage = 8
			- ServiceMessage = 9
			- ActionMessage = 10
			- ReactionMessage = 11
			- UnknowBehaviourMessage = 12,
			- ProfileUpdateResultMessage = 14
			- ActionResultMessage = 16
			- ReactionResultMessage = 18
			
	Les paquets envoyés par le client sont les suivants:

			- IdentificationMessage = 2
			- RegisterMessage = 4
			- RegisterServiceMessage = 6
			- DeleteServiceMessage = 7
			- ProfileUpdateRequestMessage = 13
			- ActionRequestMessage = 15
			- ReactionRequestMessage = 1
			
#### Communication ####

### Pour utiliser une action ###

	Requête:

        - int ActionId
		- string Params
        - string Token

		http://127.0.0.1:8080/actionrequestmessage?actionid={actionid}&token={token}&params={params}

	Réponse:
		
		- Non nul:

			public ActionResultEnum Result

			public int ActionId

			public int ServiceId

			public string Output

			public string Param

	Données:
	
		public enum ActionResultEnum
		{
			None,
			BadParams,
			Success
		}
### Pour se connecter ###

	Requête:

		- Username
		- Password

		http://127.0.0.1:8080/identificationmessage?username={username}&password={password}

	Réponse:
		
		- Non nul:
			
			public IdentificationResultEnum Result

		- Possiblement nul:

			public string Username
			public string Name
			public string Mail
			public string Token
			public List<ServiceMessage> Services

		public enum IdentificationResultEnum
		{
			None = 0,
			Success = 1,
			BadCredentials = 2,
			InternalError = 3
		}
